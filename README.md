# 📊 Табличный процессор (Spreadsheet Engine)

Вычислительное ядро электронных таблиц на C++17. Поддерживает работу с текстом, числами и формулами, отслеживание зависимостей между ячейками и оптимизацию повторных вычислений.

## ⚙️ Возможности
* Сетка размером до 16384 x 16384 ячеек.
* Поддержка текста, чисел и математических формул (с префиксом `=`).
* Обработка специфичных ошибок вычислений: `#REF!`, `#VALUE!`, `#ARITHM!`.
* Автоматическое обнаружение и блокировка циклических зависимостей.

## 🏗 Архитектура и технические решения
* **Граф зависимостей и кэширование:** Реализовано отслеживание входящих и исходящих связей. Используется ленивое вычисление (lazy evaluation) с флагом `dirty_` и кэшированием результатов, что исключает пересчет неизменившихся формул.
* **Идиома Pimpl:** В классе `Cell` применен Pimpl (`EmptyImpl`, `TextImpl`, `FormulaImpl`) для инкапсуляции внутреннего состояния и ускорения компиляции.
* **Генерация парсера (ANTLR4):** Синтаксический анализ формул реализован через ANTLR4. Строится Абстрактное Синтаксическое Дерево (`FormulaAST`), которое затем обходится для вычисления значения.
* **Интерфейсный подход:** Ядро построено на чистых интерфейсах (`CellInterface`, `SheetInterface`), что обеспечивает слабую связанность компонентов.

## 🛠 Стек
* **Язык:** C++17
* **Парсинг:** ANTLR4 (Lexer, Parser, AST)
* **Сборка:** CMake

## 📦 Сборка и запуск

Проект использует CMake и включает статическую линковку ANTLR4. Настроены флаги компиляции для MSVC (Windows) и GCC/Clang (Linux/macOS).

# Конфигурация и сборка
